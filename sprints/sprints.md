# Plano de Sprints - Migração para API Backend

Este documento organiza as sprints para a tarefa de substituir os dados mocados (`mock_data.dart`) do aplicativo Flutter pelas chamadas à API real, documentada em `api.md`.

## Sprints

*   **[Sprint 1: Refatoração e Endpoints de Visualização](./sprint1.md)**
    *   **Foco:** Configurar a base da comunicação com a API e substituir os dados de visualização (lojas, produtos e categorias).
    *   **Objetivo:** Ter o app exibindo dados reais do backend nas telas principais.

*   **[Sprint 2: Autenticação e Perfil do Cliente](./sprint2.md)**
    *   **Foco:** Implementar o fluxo de autenticação do cliente e a tela de perfil.
    *   **Objetivo:** Permitir que usuários se cadastrem, façam login e vejam seus dados.

*   **[Sprint 3: Fluxo de Pedidos e Histórico](./sprint3.md)**
    *   **Foco:** Implementar a criação de pedidos, o rastreamento e o histórico.
    *   **Objetivo:** Habilitar o fluxo principal do aplicativo: a compra de produtos.
